# Signed Token API

A RESTful API built with **Node.js**, **Express**, **MongoDB**, and **JWT** for generating and verifying time-bound signed tokens. The project also demonstrates transactional state synchronization by creating and linking related database entities atomically using MongoDB transactions.

---

## Features

- Generate cryptographically signed JWT tokens
- Configurable token expiration
- Verify token authenticity and expiry
- Transactional state synchronization
- Atomic creation of bidirectional relationships
- MongoDB integration using Mongoose
- Environment variable configuration with dotenv

---

# API Endpoints

## Generate Token

**POST**

```
/api/token
```

### Request Body

```json
{
    "expiresIn": 900
}
```

### Response

```json
{
    "success": true,
    "token": "<jwt_token>"
}
```

---

## Verify Token

**POST**

```
/api/verify
```

### Request Body

```json
{
    "token":"<jwt_token>"
}
```

### Response

```json
{
    "success": true,
    "payload": {
        ...
    }
}
```

---

## State Synchronization

**POST**

```
/api/state/sync
```

This endpoint

- Verifies the signed token
- Starts a MongoDB session
- Creates an `Entity`
- Creates a related `Directory`
- Links both documents together
- Commits all operations atomically using a MongoDB transaction

### Request

```json
{
    "token":"<jwt_token>"
}
```

---

# Transaction Flow

1. Verify JWT
2. Start MongoDB session
3. Begin transaction
4. Create Entity document
5. Create Directory document
6. Link both documents
7. Commit transaction
8. Return created resources

If any step fails, the transaction is aborted automatically.

---

# Notes

MongoDB transactions require a **Replica Set** or **MongoDB Atlas** deployment.

If MongoDB is running as a standalone server, the following error may occur:

```
Transaction numbers are only allowed on a replica set member or mongos
```

This is a MongoDB deployment requirement rather than an issue with the application logic.
